# agmsg Office

[English](README.md) | 日本語

![agmsg Office デモ](docs/agmsg-office-demo.gif)

**agmsg Office** は、[`agmsg`](https://github.com/fujibee/agmsg) のエージェント間
メッセージログを、キャラクターがステージで会話する形で再生するアプリです。各エージェントが
キャラクターになり、順番に発言する様子を眺められます。手元で起動すれば、**自分の**
エージェントの会話履歴をそのまま再生できます。ブラウザだけで動く静的な Vite + React
アプリで、バックエンドや API キーは不要です。

## クイックスタート

```bash
npm install
npm run dev
```

表示された URL を開きます。同梱サンプルが読み込まれているので、**Start** を押すと再生されます。
**自分のデータ**を見るには、**Source** ドロップダウンからローカルの `agmsg` チームを選び、
**Start** を押します。開発サーバ（`npm run dev`）の起動中は、インストール済みの `agmsg`
履歴を直接読み取って再生します。

## 仕組み

agmsg Office は、agmsg のログを読み込み、正規化し、各エージェントにキャラクターを割り当て、
メッセージを 1 件ずつ再生します。発言中のエージェントが吹き出しを出し、対応するログ行が
ハイライトされます。ホストキャラが、開始・終了やシステムイベントをナレーションします。

再生できるログは 3 種類です。

- **ローカルの agmsg 履歴** — 主な使い方です。`npm run dev` の起動中、インストール済みの
  `agmsg` データを読み取り、Source ドロップダウンに自分のチームが並びます。
- **手動でインポートする JSON ファイル** — JSON 形式で書き出した agmsg ログ。
- **同梱サンプル** — すぐ試せるデモ（英語・日本語）。

サンプルと JSON インポートはどこでも（本番ビルド `npm run build` でも）使えます。ローカルの
agmsg 履歴の読み取りだけは、ローカルの `agmsg` データを読む小さなヘルパーが動く開発サーバが
必要です。
